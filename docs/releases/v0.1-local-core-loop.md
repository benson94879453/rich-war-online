# v0.1-local-core-loop Acceptance Criteria

Baseline date: 2026-06-26

## Release Intent

`v0.1-local-core-loop` is the first local playable baseline. It does not prove production networking, final UI, intervention cards, stock market, upgrades, or reconnect. It proves that the board-game loop is stable enough to iterate on.

## Scope

In scope:

- Start `res://scenes/StarQGame.tscn`.
- Initialize four local players from the active board data.
- Roll dice and move pieces on the grid map.
- Resolve route choices.
- Resolve basic tile effects.
- Offer Buy / Skip for unowned properties.
- Apply property ownership, rent, money changes, and basic bankruptcy state already present in the prototype.
- Keep the UI, event label, money label, and log coherent during local play.

Out of scope:

- Host / Client acceptance.
- Same-seat reconnect.
- Lobby, room code, matchmaking, and accounts.
- Intervention cards and reaction windows.
- Property upgrades and stock trading.
- Visual polish beyond functional readability.

## Required Manual Scenario

Run from the Godot editor or desktop runner:

1. Open `res://scenes/StarQGame.tscn`.
2. Confirm four player pieces and four money entries are visible.
3. Play until 20 completed turns have resolved.
4. Confirm turn order repeats as P1, P2, P3, P4.
5. Resolve at least one route choice if the board path triggers one.
6. Buy at least one affordable property if offered.
7. Skip at least one property if offered.
8. Trigger rent if the dice path allows it.
9. Record any script error, stuck control state, or divergent UI state.

## Pass Criteria

- Four players are initialized and visible.
- The first visible turn is P1.
- 20 completed turns finish without script errors.
- Turn order remains P1, P2, P3, P4.
- Dice text updates on every roll.
- The active player piece moves to the displayed landing node.
- Route-choice buttons appear only during route choice and disappear after selection.
- Roll is disabled while movement, route choice, or property decision is pending.
- Buy and Skip controls appear only during property decisions and disappear after resolution.
- Money text updates after purchases, rent, and tile money effects.
- The event log records meaningful roll, landing, purchase, skip, rent, and tile-effect outcomes.
- No P0 blocker remains open.

## Failure Criteria

The run fails if any of these occur:

- The scene cannot start.
- Any player turn becomes impossible to complete.
- A route choice appears but no valid route can be selected.
- Property controls remain stuck after Buy or Skip.
- Roll becomes enabled during a pending action.
- A script error appears during the run.
